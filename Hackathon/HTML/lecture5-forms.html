Form in html

The fundamental truth is that a website isn't just a brochure for you to read. For the web to be useful, it needs a way to collect information from the user and send it back to the server.

Forms in HTML is used to accept user input. The form in html is created by using <form> element as <form>... </form>.

Without this, you couldn't log in, search for a video, buy a product, post a comment, or send a message. The web would be a read-only library.

The input Element:

  • This is the most common form tag
  • It's a self-closing tag that creates an input field.
  • Its behavior changes based on its type attribute.
  • The most basic type is text.

The label tag

Label: It's a tag specifically designed to be the title for a form field.

Problem: We have a box, but the user has no idea what they are supposed to type into it. Is it for a name? An email? A search query? The box is meaningless without a description.

Solution: We need to add a descriptive piece of text. The correct HTML tag for this is the <label>. It's a tag specifically designed to be the title for a form field.

The Need for Connection - id and for

Problem: The <label> and the <input> are still two completely separate, unrelated things.
The browser doesn't know they belong together

Solution: We need a unique naming system.

  • First, we give our input box a unique name that no other element on the page has. The attribute for a unique name is id. Let's give it an id of "firstName".
  • Next, we tell the label which element it is for. The for attribute on the label must match the id of the input.

We have now created a powerful, explicit connection.

  • If you click on the text "First Name:", your cursor will magically jump into the text box.
  • Behind the scenes: A screen reader will now announce, "First Name, edit text" when the user focuses on the input box. The two elements are now a true pair.

The Need for Data Identification - The name Attribute

Probem: When the form is submitted, the browser needs to package the data to send to a server. How does it label the data? If a user types "Arjun" in the first box, how does the server know that "Arjun" is the firstName? The id attribute is only for use within the page; it is not sent to the server.

Solution: We need another attribute whose sole purpose is to be the "data label" or the "key" for the submitted value. This is the name attribute.

Now we have a truly functional form, ready to send meaningful data.

When submitted, the browser will create a package that looks like this:
  • firstName = (whatever the user typed)
  • lastName = (whatever the user typed)

The Input Restriction are:

A list of some common input restrictions is given below, few of which can be used for validation purpose.

Attribute Description
disabled Specifies that an input field should be disabled
max Specifies the maximum value for an input field. Use the max and min attributes together to create a range of legal values.
min Specifies the minimum value for an input field.
pattern Specifies a regular expression to check the input values.

However, this attribute does not provide any feedback. So, if you want to alert the user, you must write JavaScript code.

The <button> Element

Our <input type="submit"> works, but it's very limited. You can only put plain text in it using the value attribute. What if you want a button with an image, or with bold text?

 Use the button element. It's a container tag, meaning it has an opening and closing tag. This allows you to put other HTML elements inside it.

The <select> tag :

<select> tag is used. to create drop-down list.

The attributes of <select> tag are:

  1. Name: Assigns name to the control.
  2. Multiple - It allows the user to select more than one value.
  3. Size - The size attribute is used to specify the number of visible values.

The <option> tag inside the <select> tag defines the available options in the list. The attributes of <option>tag are :

  1. selected : To define preselected option, 'selected' attribute is added to the <option>
  2. value : It assigns value to the option specified in the dropdown list.

The <textarea> tag :

The <textarea> is used to create a textbox with multiple lines. The number of lines and width of the textbox are specified by using rows and cols attribute respectively.

The <textarea>can have following attributes.

  1. name : It is used to specify name for the textarea. For example name = "ta1".
  2. rows : It specifies the number of lines in a textarea. For example rows = "5".
  3. cols : It specifies the width of a text area
  4. maxlength : It specifies the maximum number of characters allowed in the textarea.
  5. placeholder : It specifies a short hint that describes the expected value of a textarea. For example placeholder = "your address"
  6. required : It specifies that textarea must be filled out. i.e. It can not be blank.
