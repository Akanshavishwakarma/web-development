Types of Networks:

  1. PAN (Personal Area Network): Connects a small number of devices, such as a mouse, keyboard, or headphones, to a single computer or user.
  2. LAN (Local Area Network): Links devices within a limited area, like a home, office building, or school.
  3. MAN (Metropolitan Area Network): Spans a city or large campus, connecting several LANs together.
  4. WAN (Wide Area Network): Connects devices and networks over large geographical distances, such as continents or even globally, with the internet being the largest example.

Types of IP Address:

  1. IPv4

    • IPv4 (Internet Protocol version 4) is a 32-bit numerical address system that uniquely identifies devices on a network and is fundamental to the functioning of the internet and other packet-switched networks. It uses four sets of numbers, each ranging from 0 to 255, separated by periods (e.g., 192.168.1.1).

    • Problem: The main challenge with IPv4 is the limited number of available addresses, which has been insufficient for the growing number of connected devices.

    • To extend the life of IPv4, NAT technology is used to allow multiple devices on a private network to share a single public IP address, conserving the limited public address space.

    • Solution:To overcome the limitations of IPv4 entirely, IPv6 was developed, offering a vastly larger address space of 128 bits.

  2. IPv6

    • IPv6, or Internet Protocol version 6, is the latest version of the Internet Protocol that provides a vastly larger address space than the older IPv4 to accommodate the growing number of connected devices on the internet

    • Importances:
      1. Addresses IPv4 exhaustion: The primary reason for IPv6's development was to overcome the limited address space of IPv4, which was rapidly running out as more devices connected to the internet.

      2. Supports future growth: The large address pool of IPv6 is essential for the future growth of the internet, supporting everything from smartphones to smart home devices and IoT sensors.

      3. Improved network functionality: By providing more addresses, IPv6 ensures that every new device can be assigned a unique IP address, facilitating seamless communication and data transfer.

Public IP Address and Private IP Address

A public IP address is a unique global address, assigned by an ISP, that identifies your entire network to the internet, enabling communication with external services. In contrast, a private IP address is assigned by your router to individual devices (like a phone or laptop) within your local network and is only visible and used internally for communication between devices on that network. Your router acts as a gateway, using its public IP to connect to the internet while assigning private IPs to your local devices.

Features Public IP Address Private IP Address
Scope Global, identifying your network on the internet. Local to your home or office network (LAN).
Uniqueness A one-of-a-kind address for your entire home or office network on the internet. Reusable and not unique outside the private network; many devices on different networks can share the same private IP addresses.
Assignment Assigned by your Internet Service Provider (ISP) to your router. Assigned by your router to devices within your local network.
Purpose Allows devices on your network to access websites, send emails, and communicate with other devices on the internet. Enables devices within your local network to communicate and interact with each other, even without an internet connection.

DNS (Domain Name System) Server

DNS, or the Domain Name System, works by translating human-friendly domain names (like www.example.com) into machine-friendly IP addresses (like 192.0.2.1) that computers use to locate websites and other resources on the internet

DNS acts as the interface between the client and server. DNS have all website domain name with it's IP address.

DNS works:

  1. Client/User sends request to the server with Domain name. For acessing the server Client requires IP address. So, it directs to the DNS resolver.

  2. DNS resolver have 3 steps to access the IP.
    • Root Server: Resolver first contacts the Root DNS Server which is the starting point for DNS lookups. The Root server doesn't know the exact IP address of any website. It's checks for the .extension (.com, .org, .gov, .in, etc.). and send it to the TLD

    • TCD Server: Resolver sends the query to the TLD Server for .extension domains. The TLD server handles domain names ending in .extension and knows where to find the authoritative nameserver for particular website.

    • Authoritative Domain Server: The Resolver then queries the authoritative nameserver for particular website. This server is responsible for storing DNS records for the domain, including the mapping of the domain name to its IP address.

  3. After getting the IP address, now we can access the website (server).

DNS steps-by-steps Process with example:

  1. Client/User sends request to the server with Domain name. For acessing the server Client requires IP address. So, it directs to the DNS resolver.

  2. DNS Resolver:
    • Root Server:
      1. Resolver first contacts the Root DNS Server
      2. The Root server doesn't know the exact IP address of example.com but directs the query to the Top-Level Domain (TLD) Server responsible for .com.

    • TLD Server:
      1. Resolver sends the query to the TLD Server for .com domains
      2. The TLD server handles domain names ending in .com and knows where to find the authoritative nameserver for example.com.

    • Authoritative Server:
      1. The Resolver then queries the authoritative nameserver for example.com.
      2. This server is responsible for storing DNS records for the domain, including the mapping of the domain name to its IP address.

DNS types:

You'll use DNS all day everyday, but did you know there are two different forms of DNS query, recursive and iterative. Understanding how these work ensures that you understand how clients will be using your DNS server(s) and what access they might need to external DNS servers and therefore how that would affect things like firewall configuration. So let’s explore the two types:

  1. Iterative way: An iterative DNS query is a type of query where the DNS server who recieved your query won’t do the leg work, but will give you a referral to a DNS server that might have the answer; in this case your DNS client would need to do all the leg work needed to get you the answer, which basically means following all the referrals.

  2. Recursive way: A recursive DNS query is a type of query where the DNS server who recieved your query will do all the leg work needed to get you the answer (i.e. go off and ask other DNS servers) and then give you the result.

Types of ports:

  1. Well-known Port Number

    • These are the VIPs of the port world. They are reserved for the most common, fundamental, and standardized internet services.
    • Controlled By: Strictly managed by IANA.
    • Privileges: On most operating systems (like Linux and macOS), you need administrator (root or sudo) privileges to run a program that listens on these ports. This is a security measure to prevent rogue applications from impersonating standard services.

  2. Registered Ports

    • These ports are for specific applications or services that are not as universal as the "well-known" ones. Companies and developers can register a port for their software to avoid conflicts with other applications.
    • Controlled By: Less strict. You can request a port from IANA.
    • Privileges: You do not need administrator privileges to use these ports. This is the "user space" for applications.

  3. Dynamic / Private / Ephemeral Ports

    • These ports are not meant for public-facing servers. Their job is completely different.
    • Controlled By: No one. They are for temporary, private use.
    • Purpose: When your web browser (the client) makes a request to a server (e.g., to google.com on port 443), your computer's operating system needs to assign a temporary source port for its side of the conversation. It grabs a random, unused port from this high-numbered range for the duration of that single connection.
    • Key Takeaway: You never configure your server applications to listen on these ports. They are used automatically by the client side of a connection.
